nanobot Web Channel — Minimal Client

DataAgent — login or paste token

Token 不持久化：刷新或重新打开本页后，输入框为空属正常，需要重新 Login 或粘贴 token（本示例不写 sessionStorage，避免误以为「没登录却已有 token」）。 Login: browser POST to your DataAgent auth URL; the returned token is filled below and sent as metadata.auth_token on each chat message (nanobot then calls new_session / conversing with Authorization: Bearer …). If the auth server has no CORS for file:// or your dev origin, serve this HTML from the same host or use a proxy. Or paste a token manually. Optional: tools.dataAgent.authToken in config.json still applies when the field is empty.

每条消息会携带 metadata.data_agent.llm_param（仅发送非空字段；与 config.json 里 tools.dataAgent.llmParam 合并）。

WebSocket Connection

WS token 必填：与下方 DataAgent 登录返回的 Bearer 为同一 token 时，登录成功会自动填入此处；也可手动粘贴。用于 nanobot Web 鉴权与 DataAgent 请求（服务端在 auth 后会复用此 token）。

HTTP — New conversation

Enable httpApiEnabled on the Web channel and set httpApiPort (default 8766). POST clears nanobot session. If gateway sessionAuthBaseUrl is set, include auth_token in the JSON body (same as chat metadata) so the server clears web:{userId}; otherwise the legacy key web:{chat_id} is cleared.

Stream events: stream_delta / stream_end. One-shot event: message. DataAgent: root metadata.auth_token or nested metadata.data_agent (e.g. request_id/design_template_id (or camelCase), assistant, llm_param) — merged over config.json for that turn only; omit request_id to use this room’s chat_id.
